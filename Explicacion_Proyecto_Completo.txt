EXPLICACION DEL PROYECTO COMPLETO
Proyecto: Transportes Hirata


DESCRIPCION GENERAL
-------------------
Transportes Hirata es una aplicacion de escritorio desarrollada en Java Swing.
El sistema esta orientado a apoyar la gestion interna de una empresa de transporte,
permitiendo administrar camiones, conductores, usuarios, mantenimientos, equipos de
oficina, software e inventario de piezas.

El proyecto usa una arquitectura simple por capas:

- GUI: contiene las ventanas y formularios hechos con Java Swing.
- Datos: contiene los DAO, encargados de consultar e insertar informacion en la base de datos.
- Logica: contiene las clases modelo que representan las entidades del sistema.

La base de datos usada es MySQL. La conexion se realiza desde la clase Conexion y los DAO
ejecutan las operaciones SQL necesarias.


OBJETIVO DEL SISTEMA
--------------------
El objetivo principal es centralizar informacion operativa de la empresa, evitando que
los registros de camiones, conductores, mantenimientos y equipos se manejen de forma
manual o dispersa.

El sistema permite:

- Registrar camiones.
- Registrar conductores.
- Asociar conductores a camiones.
- Registrar usuarios del sistema.
- Controlar acceso por tipo de usuario.
- Registrar mantenimientos de camiones.
- Actualizar kilometraje de camiones.
- Consultar historial de mantenimiento.
- Registrar equipos de oficina.
- Registrar mantenimientos de equipos.
- Consultar historial de mantenimiento de equipos.
- Registrar actualizaciones de software.
- Controlar inventario de piezas.
- Descontar piezas usadas en mantenimientos.


TIPOS DE USUARIO
----------------
El sistema maneja permisos segun el cargo del usuario que inicia sesion.

Administrador:
Puede acceder a las funciones principales del sistema. Puede registrar camiones,
conductores, equipos, personal, mantenimientos, inventario de piezas y actualizaciones
de software. Tambien puede revisar historiales.

Personal:
Tiene un acceso mas limitado. Puede registrar mantenimientos de equipos de oficina y
actualizaciones de software, pero no puede registrar camiones, conductores, equipos,
personal, ni acceder a historiales restringidos.

Conductor:
Tiene permisos reducidos y no accede a los modulos administrativos.


MODULOS PRINCIPALES
-------------------

1. Login
El sistema inicia con una ventana de autenticacion. El usuario ingresa su RUT y clave.
Segun el cargo registrado en la base de datos, el sistema habilita o restringe opciones
en la ventana principal.


2. Registro de camiones
Permite ingresar los datos de un camion, como patente, marca, modelo, anio,
kilometraje actual y conductor asignado.

Tambien permite editar o eliminar registros, manteniendo una tabla visible con los
camiones existentes.


3. Registro de conductores
Permite registrar conductores con datos como RUT, nombre, licencia, telefono y clave.
Estos conductores pueden ser asociados a camiones.


4. Registro de personal
Permite que el administrador cree nuevos usuarios de tipo personal. Esto permite
delegar tareas operativas sin entregar permisos completos de administrador.


5. Mantenimiento de camiones
Permite registrar mantenimientos asociados a camiones. Se registra fecha, tipo,
descripcion, kilometraje y camion correspondiente.

Este modulo permite llevar control sobre las mantenciones realizadas a la flota.


6. Historial de mantenimiento de camiones
Muestra los mantenimientos registrados para los camiones. Permite revisar la informacion
historica de las mantenciones realizadas.


7. Actualizacion de kilometraje
Permite actualizar el kilometraje actual de los camiones. Esta funcion es importante
porque el kilometraje se usa como dato de control para mantenimiento y seguimiento
operativo.


8. Registro de equipos de oficina
Permite registrar equipos como computadores, impresoras y telefonos.

El tipo se selecciona desde un combo box y el codigo se genera automaticamente:

- Computador: PC-001
- Impresora: IMP-001
- Telefono: TEL-001

Esto permite identificar cada equipo de forma ordenada y uniforme.


9. Mantenimiento de equipos de oficina
Permite registrar mantenimientos de equipos de oficina. El usuario selecciona un equipo
existente y registra fecha, tipo de mantenimiento, descripcion, tecnico, observaciones
y estado.

Ademas, si el mantenimiento requiere una pieza, el usuario puede seleccionar una pieza
de inventario y la cantidad utilizada. El sistema valida stock y descuenta la pieza
automaticamente.


10. Historial de mantenimiento de equipos
Permite consultar todos los mantenimientos registrados para equipos de oficina.
Sirve para revisar trazabilidad, responsables y estado de cada intervencion.


11. Actualizacion de software
Permite registrar software disponible en un catalogo y controlar actualizaciones por
equipo.

El flujo consiste en seleccionar un equipo, seleccionar un software, ingresar version
anterior, version nueva, tecnico, estado y observaciones. Si la actualizacion se marca
como realizada, el sistema actualiza la version instalada para ese equipo.


12. Inventario de piezas
Permite registrar piezas o repuestos, indicando codigo, nombre, categoria, marca,
stock, stock minimo, ubicacion y estado.

Tambien permite registrar movimientos de inventario:

- Entrada de piezas.
- Salida de piezas.
- Ajuste de stock.

El sistema evita codigos duplicados y actualiza el estado de stock automaticamente.


BASE DE DATOS
-------------
La base de datos principal es Transportes_Hirata_v2.

Tablas principales:

- Usuario
- Conductor
- Camion
- Mantenimiento
- equipos
- MantenimientoEquipo
- Software
- SoftwareEquipo
- ActualizacionSoftware
- Pieza
- MovimientoInventario

Cada tabla representa una entidad importante del sistema y varias de ellas se relacionan
mediante claves foraneas. Por ejemplo:

- Un camion puede tener un conductor.
- Un mantenimiento pertenece a un camion.
- Un mantenimiento de equipo pertenece a un equipo.
- Una actualizacion de software pertenece a un equipo y a un software.
- Un movimiento de inventario pertenece a una pieza y puede asociarse a un mantenimiento
  de equipo.


TECNOLOGIAS USADAS
------------------
- Lenguaje: Java.
- Interfaz grafica: Java Swing.
- Base de datos: MySQL.
- Patron de acceso a datos: DAO.
- Construccion del proyecto: Maven.
- IDE esperado: NetBeans o cualquier IDE compatible con proyectos Maven Java.


ESTRUCTURA DEL CODIGO
---------------------
src/main/java/com/mycompany/transporteshirata/GUI
Contiene las ventanas del sistema. Cada archivo representa una pantalla o formulario.

src/main/java/com/mycompany/transporteshirata/Datos
Contiene las clases DAO. Estas clases realizan operaciones de base de datos como
registrar, listar, editar, eliminar y validar.

src/main/java/com/mycompany/transporteshirata/Logica
Contiene las clases modelo, por ejemplo Camion, Conductor, Equipo, Software, Pieza y
MantenimientoEquipo.


LOGICA GENERAL DEL SISTEMA
--------------------------
El usuario inicia sesion. Segun su cargo, la ventana principal habilita o bloquea menus.
Desde el menu principal se abren ventanas internas. Cada ventana permite trabajar con
una parte del sistema.

Cuando el usuario registra informacion, la GUI crea un objeto de la capa Logica y lo
envia a un DAO. El DAO ejecuta la sentencia SQL correspondiente y guarda la informacion
en MySQL.

Cuando el usuario necesita consultar datos, la GUI solicita una lista al DAO. El DAO
consulta la base de datos, transforma los resultados en objetos y la GUI los muestra
en tablas.


ESTADO ACTUAL DEL PROYECTO
--------------------------
El proyecto cuenta con los modulos centrales de gestion de transporte y ademas incorpora
un modulo completo de equipos de oficina.

Actualmente se puede controlar:

- Flota de camiones.
- Conductores.
- Usuarios y permisos.
- Mantenimiento de camiones.
- Equipos de oficina.
- Mantenimiento de equipos.
- Historiales.
- Software por equipo.
- Inventario de piezas.

Con esto, el sistema queda mas completo porque no solo administra vehiculos, sino tambien
los recursos tecnologicos internos que apoyan la operacion de la empresa.
